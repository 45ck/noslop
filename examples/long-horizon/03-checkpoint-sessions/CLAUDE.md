# Project: ShopStream E-Commerce Platform

## Product

ShopStream is a B2C e-commerce platform handling product catalog, shopping
carts, orders, payments, and fulfillment. Currently serving ~50k daily
active users with a PostgreSQL database that has grown organically over
3 years and needs significant schema restructuring.

## Tech Stack

- **Runtime:** Node.js 20 LTS
- **Framework:** Express 4 with TypeScript 5.4
- **ORM:** Prisma 5.x
- **Database:** PostgreSQL 16
- **Testing:** Vitest, Supertest for integration tests
- **CI:** GitHub Actions

## Key Commands

```bash
npm run dev          # Start dev server with hot reload
npm run build        # Compile TypeScript
npm run test         # Run unit + integration tests
npm run test:watch   # Watch mode
npm run lint         # ESLint + Prettier check
npm run db:migrate   # Run pending Prisma migrations
npm run db:generate  # Regenerate Prisma client
npm run db:seed      # Seed development database
npm run db:studio    # Open Prisma Studio
```

## Architecture

```
src/
  controllers/      # Express route handlers
  middleware/        # Auth, validation, error handling
  repositories/     # Data access layer (Prisma queries)
  services/         # Business logic
  types/            # Shared TypeScript types
  utils/            # Helpers (pagination, formatting)
prisma/
  schema.prisma     # Database schema
  migrations/       # Migration history
  seed.ts           # Dev data seeding
```

## Current Work

We are migrating the database schema to fix years of accumulated tech
debt: denormalized tables, missing indexes, inconsistent naming, and
no proper audit trail.

## Database Conventions (New Schema)

- Table names: `snake_case`, plural (`order_items` not `OrderItem`)
- Column names: `snake_case`
- Primary keys: `id` as UUID, generated by database
- Timestamps: `created_at`, `updated_at` on every table
- Soft deletes: `deleted_at` nullable timestamp where needed
- Foreign keys: `{referenced_table_singular}_id` (e.g., `order_id`)
- Indexes: named `idx_{table}_{columns}`

## Rules

- Never modify production data without a reversible migration
- All schema changes go through Prisma migrations (no raw DDL)
- Repository methods must not leak Prisma types to services
- Every repository method needs a corresponding unit test
- Integration tests use a dedicated test database, not mocks
